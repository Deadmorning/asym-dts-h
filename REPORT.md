# Asym-DTS H 版策略报告

中证 500 ETF (510500) 基于中证 500 指数 (000905.SH) 的周线+日线双层趋势跟踪策略。

---

## 一、策略核心设计

### 1.1 信号层结构

```
周线 WTS (战略层)  ←── 决定"是否允许做多"的大方向
    │
    ├── UP+UP        → 强多头周，禁用 DTS 干预（独家保护）
    ├── DOWN+DOWN    → 明示空仓（实际会被 WTS=0 先拦截）
    ├── new_bull     → 上周 WTS=0、本周 WTS=1 → 强制入场
    └── 其他 WTS=1   → 日线 DTS 双向决定

日线 DTS (战术层)  ←── 在 WTS 允许多头时，做短线买卖决策
```

### 1.2 七规则 K 线判定（WTS / DTS 共用）

把一根 K 线按振幅阈值 `FLAT` 先二分为 `FLAT` / `非 FLAT`，非 FLAT 再按收盘 vs 开盘二分为 `UP` / `DOWN`：

| Rule | 前一根 → 当前 | 判决 | 门槛 |
|---|---|---|---|
| 5 | FLAT/UP + UP 或 UP + FLAT | → 做多 | 无 |
| 6 | FLAT/DOWN + DOWN 或 DOWN + FLAT | → 空仓 | 无 |
| 1 | UP + UP | → 做多 | 幅度差 ≥ SAME |
| 2 | DOWN + DOWN | → 做空 | 幅度差 ≥ SAME |
| 3 | UP + DOWN | → 做空 | 幅度差 ≥ REV |
| 4 | DOWN + UP | → 做多 | 幅度差 ≥ REV |
| 7 | FLAT + FLAT | 维持原仓 | — |

### 1.3 H 版状态机

```python
if wts_signal == 0:
    target = 0                                  # 空仓
elif last_wts == 0 and wts_signal == 1:
    target = 1                                  # 新多头周强制入场
elif prev_class == 'UP' and curr_class == 'UP':
    target = 1                                  # UP+UP 强多头保护
elif prev_class == 'DOWN' and curr_class == 'DOWN':
    target = 0                                  # DOWN+DOWN 明示空仓
else:
    target = 1 if dts_signal == 1 else 0        # DTS 双向
```

### 1.4 参数（本轮校准最优）

| 层 | 参数 | 值 | 含义 |
|---|---|---|---|
| 周线 | W_FLAT | 0.018 | 周 K 振幅 < 1.8% 归为 FLAT |
| 周线 | W_SAME | 0.009 | 周 K 同向幅度差门槛 |
| 周线 | W_REV  | 0.012 | 周 K 反向幅度差门槛 |
| 日线 | D_FLAT | 0.007 | 日 K 振幅 < 0.7% 归为 FLAT |
| 日线 | D_SAME | 0.003 | 日 K 同向幅度差门槛（实际被 Rule 5/6 覆盖，不敏感） |
| 日线 | D_REV  | 0.003 | 日 K 反向幅度差门槛（关键参数，决定 DTS 反转灵敏度） |

## 二、H 版相对原 BigQuant v2 的结构性修正

| # | 问题（原版） | H 版修正 | 影响 |
|---|---|---|---|
| 1 | `D_SAME=D_REV=2.50` → 反向规则永不触发 | 改为 0.003 | DTS Rule 3/4 真正生效 |
| 2 | `timedelta(days=1)` 取"昨天"（日历日） | 按交易日索引取前一个交易日 | 周一不再读到周日的空数据 |
| 3 | WTS 聚合不过滤短周 | 加 `day_count >= 5` 过滤 | 节假日后 WTS 不再误切 |
| 4 | WTS 每日重算，用截至当日全量数据 | 周一一次性算完，且仅用截至"上周五"的数据 | 消除前视偏差 + 周内状态稳定 |
| 5 | `last_wts = wts_signal` 每日刷新 → 新多头周信号被吞 | 只在 **周五** 更新 `last_wts` | 保证整周 V 底捕获 |
| 6 | 状态机仅 3 分支（WTS 拦截 + new_bull + DTS） | 新增 UP+UP 保护 + DOWN+DOWN 明示 | 最大 alpha 来源 |

## 三、回测验证（2023-04-17 ~ 2026-04-15，10 万初始资金）

### 3.1 H 版主力数字

```
W_FLAT=0.018  W_SAME=0.009  W_REV=0.012
D_FLAT=0.007  D_SAME=0.003  D_REV=0.003

总收益       : +64.80%
交易笔数     : 97 笔 (约 48 买 / 49 卖)
年化近似     : ~18% (三年复合)
相对纯 WTS   : +13.33pp
```

### 3.2 DTS 网格稳健性（固定 WTS 参数）

`D_REV = 0.003` 在多个 `D_FLAT/D_SAME` 组合下都稳定落在 64.80% ± 0.01pp，说明是一个**结构性最优点**而非过拟合：

```
Top 15 参数组合全部集中在 D_REV = 0.003
D_FLAT ∈ {0.003, 0.005, 0.007}  D_SAME 不敏感
Bottom 5 集中在 D_REV ≥ 0.008 → DTS 反转太迟钝，收益下滑到 46%
```

### 3.3 关键交易年度

| 区间 | 代表交易 | 盈亏 |
|---|---|---|
| 2024-02 V 底 | 02-06 BUY 4.483 → 02-20 SELL 5.186 | **+14,621** |
| 2024-10 黑天鹅 | 10-09 BUY 6.041 → 10-15 SELL 5.655 | **-7,749** |

注：2024-10-09 是趋势跟踪不可避免的黑天鹅（情绪暴涨后暴跌），所有变体都救不了（L 版、UP+UP 放 DTS 出场版都试过）。

## 四、四方策略对比

| 版本 | 状态机关键差异 | 收益 | 备注 |
|---|---|---|---|
| **纯 WTS 基线** | 无 DTS 保护 | 51.47% | Route 1 |
| **BigQuant 原版** | 新多头强入 + DTS 双向，但两个 bug | ~15-20%（仿真） | D_REV=2.50 / 前视偏差 / last_wts 日更 |
| **GitHub asym_v2** | 新多头强入 + DTS 双向 | 40.39% | 设计上等同修好的 BigQuant，但没有 UP+UP 保护 |
| **Route 2 全对称 DTS** | 新多头 + UP+UP + DTS 双向 | 64.80% / 69.78% | **H 版，本报告主方案** |
| **Route 2 非对称版** | DTS 只出场不入场 | 43.35% | 验证失败 |
| **Route 2 部分非对称** | UP+UP 保护 + 仅 FLAT±UP 允许 DTS 双向 | 48.03% | 验证失败 |
| **Route 2 UP+UP DTS 出场版** | UP+UP 持仓中允许 DTS 出场 | 34.38% | 日内暴涨后暴跌 T+1 救不回，反而磨损加剧 |
| **Route 2 L 版（DTS 确认入场）** | 新多头周需 DTS=1 确认才入场 | 30.25% | 严重漏掉 2024-02 V 底，单笔差 -16,599 |

### 4.1 Alpha 归因分解

```
H 版 64.80%
  ├── 纯 WTS 基线       : 51.47%
  ├── + DTS 双向保护    : +13.33pp  ← Route 2 相对 Route 1 的改进
  │
  └── UP+UP 保护贡献    : 对比 GitHub 无 UP+UP 版 40.39%
                         净收益差 ≈ +24.4pp（在不同参数下）
```

### 4.2 为什么 H 版每个分支都是必要的

我们做了完整的消融实验：

1. **去掉 UP+UP 保护** → 收益跌到 40–48%，因为强多头周被 DTS 日内洗掉
2. **去掉 new_bull 强入场**（L 版）→ 收益跌到 30%，V 底全丢
3. **把 DTS 改成只出不进**（非对称）→ 收益跌到 43%，震荡市套利被阉割
4. **UP+UP 放开 DTS 出场**（试图躲黑天鹅）→ 收益跌到 34%，T+1 机制下救不了暴涨后暴跌，反而多扫磨损

结论：**H 版的每一个分支都被实验证明是必要的，且两个关键 alpha 来源（new_bull 强入 + UP+UP 保护）不可互换**。

## 五、风险与局限

### 5.1 已知无法躲的风险
- **单日极端黑天鹅**（如 2024-10-08 情绪暴涨 + 10-09 开盘进场 + 后续暴跌）：T+1 机制下 DTS 来不及救。
- **周线趋势未破但日线震荡加剧**：H 版 UP+UP 保护下可能吞掉 1-2 笔短期亏损。

### 5.2 样本外注意事项
- 参数在 2023-04 ~ 2026-04 这段三年期上校准，包含 2024 极端行情。
- 回测数据含 2026-01 到 2026-04，其中 2026-04-19 为"今天"，2026-04 之后的数据是**未来数据**，不能用于实盘优化。
- 建议实盘滚动窗口：用过去 24-36 个月校准 D_REV，其他参数（D_FLAT、W_REV）在合理区间内相对稳定。

### 5.3 实盘注意
- **前视偏差不是这个策略的 alpha 来源**。H 版严格用"截至上周五"的数据做 WTS，实盘完全等价。
- DTS 每日用"昨日盘后"算出的信号，今日按开盘价下单，与回测严格对齐。
- 手续费假设：买 3bp，卖 13bp，最低 5 元/笔（A 股 ETF 典型配置）。
